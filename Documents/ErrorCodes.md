# 错误码定义

本文档定义API返回的错误码和错误处理方式。

---

## HTTP状态码

| 状态码 | 名称 | 说明 | 常见场景 |
|--------|------|------|----------|
| `200` | OK | 成功 | 请求处理完成 |
| `400` | Bad Request | 参数错误 | 无效参数、格式错误、业务规则违反 |
| `401` | Unauthorized | 未认证 | Token缺失、Token过期、无效Token |
| `403` | Forbidden | 无权限 | 权限不足（非管理员尝试管理员操作） |
| `404` | Not Found | 资源不存在 | 文件不存在、用户不存在 |
| `500` | Internal Server Error | 服务器错误 | 内部异常、数据库错误 |

---

## 错误响应格式

### 标准格式

```json
{
  "error": "错误描述信息"
}
```

或

```json
{
  "message": "错误描述信息"
}
```

### 多错误格式

```json
{
  "errors": [
    "错误1",
    "错误2"
  ]
}
```

## 文件验证错误

### 5层验证错误

| 层次 | 验证类型 | 可能错误 |
|------|----------|----------|
| 1 | 结构验证 | `JSON解析失败: {错误信息}` |
| 2 | 大小验证 | `文件大小超出限制（最大{N}MB）` |
| 3 | 内容验证 | `文件必须包含actions数组` |
| 4 | XSS验证 | `文件包含危险的XSS内容` |
| 5 | 哈希验证 | （计算失败时内部错误） |

## 错误处理建议

### 客户端处理流程

```
1. 检查HTTP状态码
   ├─ 200 → 成功，解析响应体
   ├─ 400 → 参数错误，显示error/message
   ├─ 401 → 未认证，跳转登录
   ├─ 403 → 无权限，显示权限不足提示
   ├─ 404 → 不存在，显示404页面或提示
   └─ 500 → 服务器错误，显示通用错误提示

2. 解析错误响应
   └─ 提取 error 或 message 字段显示给用户
```

### 错误重试策略

| 错误类型 | 重试策略 |
|----------|----------|
| 网络超时 | 可重试，建议3次，指数退避 |
| 500错误 | 可重试1次 |
| 400错误 | 不重试，需修正请求 |
| 401错误 | 不重试，需重新登录 |
| 403/404错误 | 不重试，需修正操作 |

## 日志记录

服务器端会记录以下错误日志：

### 级别

| 级别 | 场景 |
|------|------|
| `Information` | 正常操作（登录、上传、审核） |
| `Warning` | 非关键错误（下载计数更新失败） |
| `Error` | 严重错误（异常、事务失败） |

### 日志格式

```
[时间] [级别] [类名] 消息
异常堆栈（如有）
```

示例：
```
2024-03-15 14:30:00 [Error] TacticsInteractionService 点赞操作失败: UserId=123, FileId=456
System.Exception: 数据库连接超时
   at TacticsInteractionService.ToggleLikeAsync(...)
```
