## 文件状态 (FileStatus)

| 状态 | 值 | 说明 | 可执行操作 |
|------|-----|------|------------|
| `pending` | 待审核 | 新上传文件初始状态 | 管理员审核、上传者删除 |
| `approved` | 已通过 | 审核通过，公开可见 | 下载、点赞、收藏、评论 |
| `rejected` | 已拒绝 | 审核未通过 | 上传者可重新上传版本 |

### 状态流转

```
上传 → pending → approved (通过)
                 → rejected (拒绝)
                      ↓
               重新上传版本 → pending
```

## 种族代码 (Races)

| 代码 | 英文名 | 中文名 | 说明 |
|------|--------|--------|------|
| `P` | Protoss | 神族 | 艾尔神族 |
| `T` | Terran | 人族 | 人类联邦 |
| `Z` | Zerg | 虫族 | 异虫群 |

### 识别规则

从 `.tactix` 文件中 `actions` 数组的第一个元素的 `itemAbbr` 字段前缀识别：

```
itemAbbr 以 'P' 开头 → 神族 (P)
itemAbbr 以 'T' 开头 → 人族 (T)
itemAbbr 以 'Z' 开头 → 虫族 (Z)
```

**示例：**
```json
{
  "actions": [
    { "itemAbbr": "Pylon", "time": 0 }
  ]
}
```
P 开头 → 神族

```json
{
  "actions": [
    { "itemAbbr": "SupplyDepot", "time": 0 }
  ]
}
```

> **注意：** `SupplyDepot` 不以 T 开头，系统会使用其他识别方式或依赖 `race` 字段。实际识别基于单位/建筑缩写的首字母。

## 配装码 (ShareCode)

### 格式规范

- **长度**：固定8字符
- **字符集**：62进制
  - `0-9`：10个数字
  - `A-Z`：26个大写字母
  - `a-z`：26个小写字母
- **格式**：`^[0-9A-Za-z]{8}$`

### 编解码规则

**编码（ID → ShareCode）：**

```
1. 编码值 = ID - 1
2. 将编码值转换为62进制（8位）
3. 字符映射：索引 → Charset[索引]
```

**解码（ShareCode → ID）：**

```
1. 将每个字符转换为索引值
2. 计算十进制值：result = Σ(index × 62^position)
3. ID = result + 1
```

### 示例对照表

| ID | ShareCode |
|----|-----------|
| 1 | `00000000` |
| 62 | `0000000z` |
| 63 | `00000010` |
| 3844 (62²) | `00000100` |
| 218,340,105,584,896 (62⁸) | `zzzzzzzz` |

### 最大容量

- **最大ID**：218,340,105,584,896（约2.18万亿）
- **足够容量**：假设每秒上传100个文件，可使用约692年

## 文件验证规则

### 文件扩展名

- 允许：`.tactix`
- 其他格式：拒绝

### 文件大小限制

根据用户等级确定，见[用户等级权限矩阵](#权限矩阵)。

### 内容验证

1. **结构验证**：有效JSON格式
2. **大小验证**：不超过等级限制
3. **内容验证**：必需 `actions` 数组
4. **XSS验证**：无危险脚本标签
5. **哈希验证**：计算SHA256防止重复

### XSS过滤规则

检测以下危险模式：
- `<script>` 标签
- `javascript:` 协议
- `on*` 事件属性
- 其他HTML危险标签

---

## 评论规则

### 内容限制

- **最小长度**：1字符
- **最大长度**：1000字符
- **XSS过滤**：自动转义HTML特殊字符

### 嵌套深度

- **最大深度**：2层
- **结构**：顶级评论 → 回复（不能再回复）

### 软删除

- 删除评论不物理删除
- `isDeleted = true`
- 内容显示为 `"[已删除]"`
- 作者信息置空
